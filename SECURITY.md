# Security Policy

## Supported Versions

We actively support the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security very seriously. Thank you for improving the security of our
open source software. We appreciate your efforts and responsible disclosure and
will make every effort to acknowledge your contributions.

### How to Report

**DO NOT CREATE A PUBLIC ISSUE.**

Report security vulnerabilities by emailing the Woclaw Security Team at:

```
security@woclab.ai
```

### Response Timeline

You can expect:

| Timeline | Action |
|----------|--------|
| **24 hours** | Acknowledge receipt of your report |
| **48 hours** | Initial assessment and triage |
| **7 days** | Status update with remediation plan |
| **30 days** | Public disclosure and fix release |

### Disclosure Process

1. Report received and acknowledged
2. Security team verifies and reproduces the issue
3. Preliminary fix developed and tested
4. CVE number requested (if applicable)
5. Fix applied and released
6. Public disclosure with credit

### Reward Program

We run a private security bounty program. If your report leads to a
significant security improvement, we may reward you at our discretion.
